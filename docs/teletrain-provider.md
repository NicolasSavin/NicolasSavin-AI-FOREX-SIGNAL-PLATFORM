# Интеграция Teletrain/Телетрэйн

Пакет `app.business_trip.providers.teletrain` содержит только адаптерную границу: конфигурацию, protocol-клиент, mock-клиент, provider, mapper, DTO и исключения. Технический id — `teletrain`; юридическое название сервиса, договор и формат API нужно подтвердить у поставщика.

Переменные окружения без секретных значений:

- `TELETRAIN_ENABLED=false`
- `TELETRAIN_BASE_URL=`
- `TELETRAIN_LOGIN=`
- `TELETRAIN_PASSWORD=`
- `TELETRAIN_TIMEOUT_SECONDS=20`

Для запуска реальной интеграции нужно получить договор, тестовый контур, официальную API-документацию, login/password или token, base URL, лимиты частоты, список методов, формат вагонов и мест, правила использования данных.

Ожидаемые операции клиента: поиск поездов, получение информации о вагонах, получение доступных мест, получение типов мест. Mapper нормализует данные поставщика в `RailwayPlace` и `RailwayCarriageAvailability` без сохранения полного сырого ответа.

Система не выполняет scraping, не угадывает закрытые endpoints, не хранит секреты в GitHub и не бронирует билеты.
