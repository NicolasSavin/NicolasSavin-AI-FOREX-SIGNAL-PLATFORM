# Интеграция «Мой Агент»

Пакет `app.business_trip.providers.myagent` содержит только интеграционную границу: конфигурацию, protocol-клиент, mock-клиент, provider, mapper, DTO и исключения. Реальные HTTP endpoints не реализованы, потому что нужны официальные credentials и документация поставщика.

Переменные окружения без секретных значений:

- `MYAGENT_ENABLED=false`
- `MYAGENT_BASE_URL=`
- `MYAGENT_LOGIN=`
- `MYAGENT_PASSWORD=`
- `MYAGENT_TIMEOUT_SECONDS=20`

Для подключения нужно запросить у поставщика договор, тестовый контур, API-документацию, login/password или token, base URL, ограничения частоты, список методов, формат вагонов и мест, правила использования данных.

Ожидаемые операции клиента: поиск поездов, получение вагонов, получение доступных мест, получение типов мест. Mapper преобразует carriage, compartment, place number, place type, gender restriction и availability в `RailwayPlace`.

Система не покупает, не бронирует и не оформляет билеты. Она только ищет маршруты и проверяет доступность мест при наличии подтверждённого источника.
